Prework Study Guide

HTML

  • HTML is used for the structure of a website
  • HTML elements are text, image, header, and lists. They can also be considered meta-related, content division, or semantic
  • head tag is primarily metadata
  • body tag is visible data
  • semantic elements are descriptive and should be used often
  • child elements are html that is nestled in a parent element eg body->main->section tag relationship

CSS

  • CSS can come in 3 forms. Inline which is in the written inside the element. This can also be written in the head element to effect everyone on the page. This is an Internal Style Sheet. External Style Sheets which has the code in a seperate file.
  • CSS rule goes: selector {property: value;}
  • selectors define what the rules apply to (elements or attributes)
  • declarations contain property to apply and the value of the property. I.e. the property and the value together
  • It is necessary to use the HTML link in the head element, so the HTML and the CSS files know about eachother
  • A class attribute allows you to stylize different elements of the same type, but differently. It'll apply to elements that are classed the same. This is done by using a .variable {declarations}
  • ELements are classed wihtin the first tag of the element. i.e. [chevron]section class="variable" [end chevron]
  • Colors can use hex codes or semantics

Git

  • git status will tell you the working branch and if there are any commits that need to be made
  • git checkout (branch) will move you to that branch
  • git checkout -b (name) will create a new branch
  • git commit -m "(message/description)" will make a new commit and the message should say what you did
  • git pull origin main will pull source code from the cloud and ensure all changes are up-to-date
  • git push origin (name-of-branch) will push changes to the cloud
  • git clone (URL) will copy a repo on the cloud onto your machine
